# brev-diary UI / UX Pass 1 Implementation Plan

> **For Hermes:** Execute in small verified steps with separate commits.

**Goal:** 修复一批真实用户可感知的问题，并把登录页与时间线首页做成更完整、更顺眼的第一版。

**Architecture:** 这次只动前端 React + CSS，不改接口协议。功能上优先修复“编辑退出丢字”和“登录态失效后卡住”两类问题；体验上优先强化登录页层级、消息反馈、时间线信息密度和移动端可读性。

**Tech Stack:** React, TypeScript, Vitest, Vite, CSS

---

## Task 1: 补回归测试，锁定编辑退出丢字问题
- 在 `frontend/src/App.test.tsx` 增加用例：输入正文后立即返回，重新打开同一篇时内容仍在。
- 验证旧实现先失败，再修复。

## Task 2: 修复编辑页退出/切换时未落盘的草稿
- 修改 `frontend/src/useDiaryApp.ts`
- 增加 pending draft flush 逻辑，在 `closeEditor` 和 `openTimelineEntry` 前尽量落盘。
- 保持预览模式与真实模式行为一致。

## Task 3: 统一登录态失效处理
- 修改 `frontend/src/useDiaryApp.ts`
- 抽出统一的 auth error 处理函数。
- 让回收站加载、恢复、永久删除等动作遇到 `missing_token` / `invalid_token` / `invalid_user` 时直接回到登录页。

## Task 4: 打磨登录页视觉与反馈
- 修改 `frontend/src/components/AuthView.tsx`
- 修改 `frontend/src/styles.css`
- 增加副标题、表单说明、成功消息样式、次级按钮样式、提交按钮 loading / disabled 态。
- 弱化“本地预览”，强化主登录动作。

## Task 5: 提升时间线首页的信息密度和移动端可读性
- 修改 `frontend/src/components/TimelineView.tsx`
- 修改 `frontend/src/styles.css`
- 给条目增加一行正文摘要。
- 改善设置菜单层级和顶部区域间距。
- 在小屏下把交错时间线压成单侧卡片流，降低阅读负担。

## Task 6: 验证、部署、发布
- `cd frontend && npm test`
- `cd frontend && npm run build`
- `docker run --rm -v /opt/brev-diary-iter1/backend:/app -w /app golang:1.26 /usr/local/go/bin/go test ./...`
- 本机部署验证后，通过 GitHub MCP 建分支 / PR。
